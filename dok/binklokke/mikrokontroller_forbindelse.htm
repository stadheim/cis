Atmel Studio med ATMega32

 

1.    Last ned USBTiny og avrdude og få forbindelse mellom datamaskinen og mikrokontrolleren:

a.    Last ned USBTiny, avrdude og eventuelt WinAVR til datamaskinen

b.    Fysisk koble datamaskinen til mikrokontrolleren

c.     Åpne Command prompt

d.    Skriv inn: avrdude -c usbtiny -p m32

e.    Du skal nå få bekreftelse at datamaskinen har forbinde med mikrokontrolleren.

OBS! Kommer en feilmelding opp må feilsøking initialiseres. Trolig er det en feil med kablingen. Begynn med å prøve å koble ut og inn igjen ledningen til mikrokontrolleren.

 

Det skal se slik ut:

 

2.    Laste ned programvareplattformen Atmel Studio 7:

a.    Søk opp "Atmel Studio 7" eller gå rett inn på microchip nettsiden for innstallation og updates for programmet.

b.    Blad ned på siden til "Downloading and Installing" og trykk på lenken for nedlastning

c.     Last ned installer-pakken og følg veiledningen videre 

 

3.    Konfigurere Atmel Studio 7 til å virke med mikrokontrolleren Atmega 32:

a.    Tools > External Tools

b.    Ett nytt vindu skal vises. Følgende data skal inn i hvert vindu:

Felt

Input

Kommentar

Title

*valgfri*

Kaller min "Release" som påminnelse for å sette på release hver gang. Dette kommer litt senere.

Command

avrdude.exe

Tar i bruk programmet avrdude som ble nedlastet tidligere 

Arguments

-c usbtiny -p m32 -U flash:w:$(ProjectDir)Release\$(TargetName).hex:i

Argumentsetning for at avrdude skal forstå hva den skal gjøre ved «Release»

Initial directory

$(BinDir)

Gjør at man kan kalle mappen hva man vil 

 

Ender opp med å se noe slik ut:

c.     Huk av "Use Output window" for å få opp feilmeldinger og diverse i Output vinduet nederst i atmel programmet. Dette er ikke kritisk, men veldig praktisk.

d.    Trykk "OK" for å lagre.

 

4.    Atmel Studio 7 er nå konfigurert til å laste opp program til en Atmega32-mikrokontroller. For å laste opp program:

a.    Åpne programfilen du ønsker å laste opp til mikrokontrolleren

b.    Build > Build Solution

c.     Sørg for at det står "Release" i Rullemenyen oppe på verktøy linjen før neste steg. Neste steg vil ikke fungere om den står på "Debug".

d.    Tools > *valgfritt navn på external tool" (Den som her ble kalt "Release")

e.    Output vinduet skal nå vise melding fra avrdude at filen ble suksessfullt lastet opp til mikrokontrolleren
